HTML/CSS/JS

PHP

I don't get all the fuss about PHP. I really like PHP 7, and enjoy it's strict argument typing and performance improvements. Using a framework like Laravel or Symfony on top makes the language even better to prototype in, and - if given the choice - would prefer to make a web-app in PHP than ASP.NET.

Laravel

I love using this PHP framework for building REST-ready web applications, and it makes my code more readable and easier to document. You can find one project on my GitHub that's done entirely in Laravel.

C#

I've done a lot of work in this language, mostly in the .NET framework but some involving Mono on Unity and Xamarin.

ASP.NET

Most of my previous work experience involved some form of ASP.NET, whether it be creating a new web application (usually MVC) from the ground up, or adding new features to an existing project.

Xamarin

Isn't it everyone's dream to build an iPhone or Android app? I considered learning Obj-C (and potentially now the new Swift) as well as Java to accomplish this, but I found leveraging my C# knowledge using Xamarin to build for both platforms at the same time to be much faster.

C++

The first language I learned anything in is also the language I primarily use at my current university job. Everyone's got their personal C++ variant, mine is C++11. No, I don't need libBoost to get anything done either.

GTK+ Framework

When building a GUI on Linux, you're kind of herded into two choices: Qt or the open-source GTK framework. Seeing as Qt charges for commercial use, at work I've often opted to program GUI applications in C or C++ in this somewhat large framework.

Python

Ah, the language where everything is pseudocode. I've used Python (specifically Python 3) for a lot of university projects and some AI competitions I've participated in. Although my heart is still with C++, I will often use Python to prototype or create a proof-of-concept.

Linux/UNIX

Some days it takes me twice to type "sudo" correctly, but the other days are much better. I can navigate around a terminal and have a pretty intermediate knowledge of Linux and POSIX compliant systems as well. Some would call me a heathen, but I prefer nano.

Windows Server

Might as well learn the other side of the server-software coin. I've got plenty of experience maintaining Active Directory and Group Policy, as well as running a remote gateway host and some of the fancier Microsoft-esque features. I'm OK at PowerShell (with a handy cheat-sheet close by) when necessary.

Video Production

I know my way around Premiere Pro and After Effects, and can certainly edit and post-process a video. While I can't do any of the crazy stuff you've probably seen in After Effects, I can walk clients through the production process and shoot and edit video as necessary.
